# Чек-лист: Что Не Готово (Текущие Гэпы)

Дата: 2026-02-22

Документ фиксирует реальные “дыры” и зоны риска, которые надо закрыть, чтобы проект был предсказуемым в разработке и проде.

## P0 (блокеры / прод-риски)

- **Email/уведомления**: `backend/src/modules/notifications/tasks.py` сейчас **stub** (печатает в stdout), нет реальной отправки через SMTP/провайдера.
- **Prod без Celery**: в `docker-compose.prod.yml` сейчас нет `celery_worker`/`celery_beat`. Любые фоновые задачи (уведомления, периодика) в проде не будут работать.
- **Расписание и повторы**: `recurrence` хранится (теперь и RRULE можно), но выдача по диапазону **не “разворачивает” повторяющиеся события** в occurrences. Это функциональный гэп.
- **AI лимиты vs тарифы**: лимиты завязаны на `Organization.plan`. Нужно убедиться, что:
  - план реально меняется там, где ожидается (billing flow / супер-админ),
  - лимиты совпадают с UX и текстами на странице тарифов,
  - корректно считается “сегодня” (UTC vs local) и пользователю понятно, по какой зоне считается лимит.
- **AI provider контракт**: приложение ожидает OpenAI-совместимый `/v1/chat/completions`. Любые отличия у провайдера (Timeweb Agent) нужно покрыть адаптером + тестами (401/403/429/5xx, отсутствие `usage`, streaming если появится).

## P1 (высокий приоритет)

- **AI: создание таблиц/колонок/записей**: поведение должно быть детерминированным.
  - Большие “пакеты данных” должны создаваться батчами (не вываливать 50-100 записей одним payload).
  - Нужно больше серверной валидации экшенов (минимальные поля, лимиты на размер payload, понятные ошибки).
- **AI: выбор типа виджета**: сейчас UI-выбор фиксируется для “пустых/metric” виджетов. Нужно расширить логику на случаи, когда модель возвращает виджеты без нужных полей или вообще без списка виджетов (чтобы UI-выбор всегда соблюдался).
- **UI/адаптация (Reports/AI/Knowledge Base)**: требуется проход по брейкпоинтам (lg/xl/2xl), чтобы не было “прыжков” и перекрытий панелей.
- **Superadmin UX**: привести к “видно, что происходит”:
  - пустые состояния,
  - ошибки (не настроен супер-админ, нет прав),
  - понятные сообщения о том, где лежат конфиги (secrets.yml).
- **Monitoring value**:
  - Prometheus/Grafana сейчас показывают базовые метрики. Нужны прикладные метрики (ошибки, latency, rate-limit hits, AI usage, export, schedule).
  - `node-exporter` в Docker Desktop показывает “внутренний взгляд контейнера”, не хостовые метрики. Для настоящего хоста надо менять режим/маунты.

## P2 (техдолг / улучшения)

- **EntryPoint сообщение**: `backend/entrypoint.sh` текст про “migrations run inside app startup” устарел (миграции вынесены в `bootstrap`).
- **Проверка безопасности для prod**:
  - `TrustedHostMiddleware`,
  - ограничение размера тела запроса,
  - строгие cookie flags (если будут cookie),
  - hardening заголовков (CSP и т.д.) и HTTPS-only режим.
- **Billing**:
  - проверить end-to-end (оплата -> webhook -> смена тарифа -> лимиты AI),
  - добавить sandbox режим и документацию.
- **Документация**:
  - README уже описывает сервисы/контракты, но нужен отдельный “Runbook” (как дебажить 500/422, как смотреть логи, как мигрировать, как менять тариф/лимиты).
