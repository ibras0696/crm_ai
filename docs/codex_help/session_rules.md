# Session Rules (Strict)

- Работать аккуратно и системно, без костылей.
- Если есть сомнение по логике: сначала читать код и проверять фактическое поведение.
- Не делать предположения о критичных участках без проверки в проекте.
- При правках сохранять архитектурный стиль проекта и текущие соглашения.
- Избегать регрессий: проверять тестами/валидацией после изменений.
