## Спринт 1 (P0): Надежность И Безопасность

- Сейчас: `backend/src/modules/notifications/tasks.py` печатает в stdout.
- Сделать:
  - SMTP отправку (минимум) с таймаутами/ретраями.
  - нормальные ошибки + логирование.
- DoD:
  - Invite email реально уходит.
  - Отправка вынесена в Celery, не блокирует API.

### S1-T2: Prod включает Celery worker/beat
- Сейчас: в `docker-compose.prod.yml` нет `celery_worker`/`celery_beat`.
- Сделать:
  - добавить сервисы в prod compose или отдельный деплой-манифест.
- DoD:
  - фоновые задачи реально выполняются в prod окружении.

### S1-T3: Access Rules реально ограничивают доступ (а не “для галочки”)
- Сейчас:
  - `backend/src/modules/access/routes.py:check_access` нигде не используется.
  - default policy: `return True` (allow).
- Сделать:
  - вынести `check_access` в сервис/dep.
  - подключить к модулям: `tables`, `knowledge`, `reports`, `schedule`, `files` (read/write/delete).
  - определить политику по умолчанию:
    - рекомендовано: **deny-by-default** для не `owner/admin`.
- DoD:
  - правило типа “employee can_write=false for table” реально блокирует PATCH/POST.
  - есть тесты на deny/allow.

### S1-T4: Приглашения сотрудников (invite token delivery)
- Сейчас:
  - `POST /orgs/invites` не возвращает `token` (это правильно).
  - нет гарантированного канала доставки токена (email stub).
- Сделать:
  - отправка письма с invite-link (или dev-only “скопировать ссылку”).
  - endpoint “resend invite”.
  - защита от повторов/спама (rate-limit).
- DoD:
  - админ может пригласить сотрудника, сотрудник принимает по ссылке без доступа к БД.

### S1-T5: Billing -> План -> Лимиты AI (end-to-end)
- Риск: UI/админка меняют лимит, но “эффективный” план не меняется или наоборот.
- Сделать:
  - четко определить “источник правды” по плану: `organizations.plan` или `subscriptions.plan`.
  - при оплате/смене тарифа обновлять то поле, которое реально используется лимитером.
  - добавить интеграционные тесты.
- DoD:
  - смена тарифа отражается в `GET /api/v1/ai/status` и в реальном лимите.

## Спринт 2 (P1): Функциональные Гэпы

### S2-T4: Экспорт/импорт: стабильность и тесты
